# 해외 축구 통계 서비스 및 데이터 공급자 조사

조사 기준일은 2026-07-05이다. 가격과 커버리지는 자주 바뀌므로 계약 직전에 다시 확인해야 한다.

## 1. 화면과 UX를 벤치마킹할 서비스

### FotMob

경기 화면에서 평점, 득점·도움, 슈팅과 xG/xGOT, 패스·기회 창출, 드리블·터치·경합, 태클·가로채기·클리어링·회복 등의 선수 통계를 보기 좋게 묶는다. 선수/대회 페이지에는 합계와 90분당 수치, 슈팅 맵, 폼과 순위를 결합한다.

좋은 점:

- 일반 팬이 이해하기 쉬운 공격/패스/점유/수비 구분
- xG, xGOT, xA와 실제 결과를 한 화면에서 비교
- 90분당 수치와 대회 순위로 맥락을 제공
- 대한민국처럼 관심 팀·선수를 빠르게 파고드는 동선

주의점:

- 공식 공개 개발자 API가 확인되지 않는다.
- 사이트 하단은 robots, crawler, indexing 등 자동화 서비스와 체계적·정기적 이용을 허용하지 않는다고 명시한다.
- 제품 UX만 참고하고 내부 엔드포인트 수집은 하지 않는 편이 안전하다.

참고:

- https://www.fotmob.com/leagues/77/stats/world-cup/players
- https://www.fotmob.com/term-of-service

### Sofascore

실시간 평점과 상세 선수 통계 외에 히트맵, 슈팅 맵, 패스 맵, 드리블 맵, 수비 맵, 평균 위치, 공격 모멘텀처럼 위치·흐름을 시각적으로 전달하는 능력이 강하다. 평점은 3.0~10.0이고 경기 중 반복 갱신되며, 득점·도움·빅찬스 창출 같은 긍정 이벤트와 퇴장·자책골·실점 유발 실수 같은 부정 이벤트를 반영한다고 설명한다.

대표 선수 통계 묶음:

- 공격: 골, 도움, 슈팅, 유효 슈팅, xG/xGOT, 빅찬스 성공·실패
- 패스: 정확한 패스, 패스 성공률, 키패스, 크로스, 롱패스
- 점유/경합: 터치, 볼 소유 상실, 드리블, 지상·공중 경합, 피파울
- 수비: 태클, 가로채기, 클리어링, 블록, 볼 회복, 드리블 허용
- 골키퍼: 선방, 박스 안 선방, 실점, 페널티 선방 등
- 위치 데이터: 히트맵과 패스·드리블·수비 행동 좌표

주의점:

- 공개 개발자용 API가 아니다.
- 이용약관은 상업적 이용, 자동 요청, aggregation, scraping, reproduction 및 데이터베이스의 상당 부분 추출을 명시적으로 제한한다.
- 평점을 제3자 화면에 표시할 때도 별도 브랜드 가이드의 로고, 링크, 색상 규칙이 있다. 허용 범위와 데이터 제공 계약은 별개로 확인해야 한다.

참고:

- https://corporate.sofascore.com/about/rating
- https://www.sofascore.com/rating-guidelines
- https://www.sofascore.com/news/introducing-sofascores-new-maps-a-smarter-way-to-read-the-game
- https://www.sofascore.com/sv/terms-and-conditions

### WhoScored

오랫동안 Opta 이벤트 데이터를 바탕으로 한 자동 평점, 경기별 선수 수치, 히트맵, 팀/선수의 강점·약점과 플레이 스타일을 결합해 왔다. 특히 수치에서 “공중볼에 강함”, “직접 프리킥에 강함” 같은 서술형 특성을 만들어 주는 방식이 참고할 만하다.

대표 지표는 출전 시간·포지션·평점, 골·도움·슈팅, 키패스·패스 성공률, 드리블, 태클·가로채기·클리어링, 공중볼, 파울·카드 등이다. 평점 알고리즘의 정확한 가중치는 공개되지 않은 독점 모델로 봐야 한다.

주의점:

- 개발자용 공개 API로 제공되는 서비스가 아니다.
- 현재 자동 접근에 403/봇 차단이 적용된다. 우회 수집을 제품 기반으로 삼지 않는다.
- 평점과 서술형 특성을 그대로 복제하지 말고, 계약 데이터로 우리만의 설명을 만든다.

참고:

- https://www.whoscored.com/
- https://theanalyst.com/articles/opta-football-stats-definitions

### FBref / Stathead

검색과 표 중심으로 선수·팀·대회·역사 데이터를 비교하기 좋다. 기본 통계는 경기, 출전 시간, 득점, 카드 등을 포함하고 중간 수준 통계에는 슈팅, 크로스, 태클 승리, 가로채기 등이 포함된다.

중요한 현재 상태:

- 2026년 초 기존 데이터 공급자와의 관계 변화로 고급 축구 데이터가 제거되었고, 현재 화면과 커버리지는 이전 자료를 전제로 설계하면 안 된다.
- Sports Reference는 자사 데이터를 스크래핑해 사이트나 도구를 만드는 것을 허용하지 않는다고 명시한다.

따라서 UX와 데이터 테이블 설계의 참고처이지, 서비스 데이터 원천으로 쓰기 어렵다.

참고:

- https://fbref.com/en/stathead/stat_coverage.cgi
- https://fbref.com/en/
- https://fbref.com/data_use.html

### Understat

xG에 집중한 분석 서비스다. 슈팅별 위치와 xG, 선수의 xG·xA·npxG, xGChain, xGBuildup을 탐색하는 경험이 강점이다.

- xGChain: 슈팅으로 끝난 한 점유 과정에 관여한 선수에게 그 슈팅의 xG를 연결한 값
- xGBuildup: xGChain에서 슈터와 마지막 패스 제공자를 제외해 빌드업 기여를 보려는 값

한계:

- 공개 공식 API가 아니라 커뮤니티 라이브러리는 HTML/내장 JSON을 수집하는 래퍼다.
- 주로 유럽 6개 리그에 초점이 있어 2026 월드컵 원천으로 맞지 않는다.
- 고급 지표 UX 연구용으로만 사용한다.

참고:

- https://understat.com/
- https://understat.com/player/5249

### Sportmonks에서 확인되는 선수 경기 통계

`fixtures/{id}?include=lineups.details.type` 형태로 참가 선수의 상세 수치를 가져오는 구조다. 문서에는 다음 유형이 확인된다.

- 기본: 선발/교체, 주장, 포지션, 출전 시간, 평점
- 공격: 골, 도움, 전체/유효/빗나간/블록 슈팅, 골대, 오프사이드, 빅찬스 창출·실패, xG
- 패스: 시도·성공·성공률, 키패스, 크로스, 롱패스, 스루패스
- 점유/경합: 드리블 시도·성공, 볼 빼앗김, 전체 경합·승리, 공중 경합 승리
- 수비: 태클, 가로채기, 클리어링, 드리블 허용, 슛 블록, 실점 유발 실수
- 골키퍼: 선방, 박스 안 선방, 실점
- 규율: 파울·피파울, 경고·퇴장, 페널티

모든 대회·경기에 모든 필드가 있다는 뜻은 아니다. 실제 월드컵 fixture로 null 비율과 갱신 속도를 측정해야 한다.

### API-Football의 특별한 권리 주의점

가격은 매우 매력적이지만 약관은 다음을 분명히 한다.

- 데이터 자체의 재판매는 금지하지만 앱, 웹사이트, 판타지 게임을 만드는 용도는 예시로 든다.
- 동시에 API-Football이 대회 데이터의 사용·게시 라이선스를 부여하는 것은 아니며, 리그·연맹·대회 주최자에게 필요한 허가를 받는 책임은 사용자에게 있다고 명시한다.
- 로고·선수 이미지·상표에도 별도 권리가 있을 수 있다.

따라서 구독 결제만으로 상업적 재게시 권리가 완성됐다고 가정하면 안 된다.

참고:

- https://www.api-football.com/pricing
- https://www.api-football.com/coverage
- https://www.api-football.com/terms

### StatsBomb Open Data의 올바른 위치

공개 저장소는 연구와 축구 분석에 대한 관심을 위해 일부 대회를 공개한다. 이를 기반으로 연구·분석·인사이트를 배포하면 StatsBomb 출처와 로고를 표시하라고 요구한다. 최신 상용 월드컵 서비스 피드의 대체물이 아니라 다음 용도로 적합하다.

- 이벤트 좌표 파서 개발
- 슈팅 맵/패스 맵 UI 프로토타입
- 공급자 어댑터 및 집계 로직 테스트
- xG 설명 콘텐츠의 예시 제작

참고:

- https://github.com/statsbomb/open-data
- https://statsbomb.com/what-we-do/soccer-data/

## 3. 서비스별 UX에서 가져올 것

| 문제 | 좋은 참고 사례 | 우리 서비스에 적용할 방식 |
|---|---|---|
| 숫자가 너무 많음 | FotMob의 범주화 | 기본 8개 지표를 먼저 보여주고 공격/패스/수비를 펼침 |
| 평점만 맹신함 | Sofascore 평점 설명 | 평점 공급자·산식 비공개 여부와 주요 영향 이벤트를 표시 |
| xG가 어려움 | Understat 슈팅 맵 | 각 슛의 위치·xG와 “100번이면 평균 몇 골” 설명 |
| 수비 기여가 안 보임 | Sofascore 수비 맵 | 태클·가로채기·클리어링·회복 좌표를 색으로 구분 |
| 표가 맥락 없이 큼 | FBref의 per-90/필터 | 합계, 90분당, 대회 백분위를 명확히 구분 |
| 숫자의 의미가 불명확 | Opta 정의 사전 | 지표 옆 `?`에서 짧은 정의와 공급자 차이를 표시 |

## 4. 지켜야 할 원칙

- “웹에서 보인다”와 “서비스에 재게시할 권리가 있다”는 다른 문제다.
- API 키는 브라우저에 두지 않고 서버에서만 사용한다.
- 원본 응답, 정규화 값, 사용자 노출 값을 분리한다.
- 공급자, 수집 시각, 수정 시각, 지표 정의 버전을 저장한다.
- 누락 값은 0으로 바꾸지 않는다. `미제공`, `집계 중`, `0`은 서로 다르다.
- 서로 다른 공급자의 xG나 평점을 같은 척도처럼 직접 합치지 않는다.
- 팀 엠블럼, 대회 로고, 선수 사진은 통계 수치와 별도의 라이선스 항목으로 다룬다.
- 본 문서는 제품 리스크 검토이며 법률 자문은 아니다. 출시 전 데이터 계약과 대한민국에서의 표시·저작권 이슈를 전문가에게 검토받는다.
