# 데이터 공급자 계약 전 확인표

구독 버튼을 누르기 전에 영업/지원팀으로부터 이메일 또는 계약서로 답을 받아 보관한다.

## 권리

- [ ] 대한민국 사용자 대상 공개 웹사이트/모바일 앱에 수치를 게시할 수 있는가?
- [ ] 유료 구독, 광고, 제휴가 붙은 상업 서비스도 허용되는가?
- [ ] 공급자 이름/로고 표기가 필수인가? 정확한 문구와 위치는?
- [ ] 원본 수치를 한국어로 번역·분류하고 자체 그래프로 표현할 수 있는가?
- [ ] 공급자 평점 이름과 색상 체계를 표시할 수 있는가?
- [ ] API 응답을 우리 DB에 영구 저장하고 계약 종료 후 보존할 수 있는가?
- [ ] 과거 경기 backfill과 사용자 다운로드/공유 범위는?
- [ ] 데이터에서 계산한 90분당, 백분위, 자체 평점 등 파생 통계를 게시할 수 있는가?
- [ ] 팀 엠블럼, 대회 로고, 국기, 선수 사진 권리가 데이터 계약에 포함되는가?
- [ ] FIFA/대회/협회에 별도로 받아야 하는 라이선스가 있는가?

## 데이터

- [ ] 2026 월드컵 104경기와 모든 참가 선수의 커버리지가 보장되는가?
- [ ] 경기별 선수 통계의 정확한 필드 목록과 정의 사전을 받을 수 있는가?
- [ ] xG, xA, xGOT, 평점, 슈팅 좌표, 패스 좌표가 포함되는가?
- [ ] 선발, 벤치, 교체, 포메이션 좌표가 포함되는가?
- [ ] 라이브 값이 몇 초마다 갱신되는가?
- [ ] 경기 종료 후 값이 언제 final이 되며 정정 알림/webhook이 있는가?
- [ ] 결측과 0을 어떻게 구분하는가?
- [ ] 선수·팀 ID는 시즌이 바뀌어도 안정적인가?
- [ ] 대한민국 선수 이름과 한글명이 제공되는가? 아니라면 원문 사용 권한은?
- [ ] 데이터 품질 SLA와 장애 보상 조건은?

## 기술과 비용

- [ ] 초당/분당/시간당/일당 제한을 모두 확인했는가?
- [ ] 라이브 경기 수가 늘 때 필요한 요청 수를 계산했는가?
- [ ] 429에 `Retry-After`, ETag, webhook 또는 delta feed가 있는가?
- [ ] sandbox나 과거 실제 경기 fixture가 있는가?
- [ ] 초과 과금인가, 차단인가?
- [ ] 월드컵 이후 리그 추가 가격과 과거 데이터 가격은?
- [ ] API 버전 종료 사전 통보 기간은?
- [ ] 계약 해지 시 데이터 삭제 의무와 유예 기간은?

## 월드컵 MVP 수용 기준 예시

- 핵심 스코어/이벤트 p95 지연 20초 이하
- 선발 발표 후 3분 이내 반영
- 출전 선수의 핵심 통계 수신율 99% 이상
- 최종 통계 정정 여부를 다음 날까지 확인 가능
- 응답 스키마 변경 사전 통지
- 공개 팬 서비스 게시 권한과 파생 통계 권한이 서면으로 명시됨
